Pyg - a Python Graphing Library¶

Pyg - pronounced ‘pig’ is my own personal graphing library for python. If you haven’t read the short introduction here, you might want to. It will explain to you if you actually want to use this library for whatever you’re planning to do. There are very few caveats to reading this documentation, but something to keep in mind is that I wrote very little actual graphing for this library, just wrapped matplotlib in a way that made things easier and more repeatable (and prettier, too). Good luck and happy coding!

class twod.ah2d(env='plot', colors='purdue')[source]¶

Bases: object

A pyg.ah2d object plots many two-dimensional data types.

The ah2d class provides an access to matplotlib charting functions and some hook ins to making these functions easier to use and more repeatable. The constructor itself takes only one optional argument, env.

add_subplot(subp=121)[source]¶

ah2d.add_subplot adds a grid in which you can make subplots.

ah2d.add_subplot follows Matlab’s lead and allows you to plot several axes on one plot. The newly created axes is saved as ah2d.ax2 - this should be expanded for more axes later.

legend(loc=1, axes=None)[source]¶

ah2d.legend shows the legend on the plot.

ah2d.legend toggles the legend showing on. This is done by getting the included objects and titles from the matplotlib axis item, and then checking to see if there is the word ‘connector’ in that title. If there is that word, then the entry is discarded.

Parameters:loc (int) – The location of the legend in counter-clockwise notation.
Returns:None
